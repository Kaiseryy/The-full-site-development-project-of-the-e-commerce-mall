安装 Docker — 所有服务的「集装箱」

📦 Docker 是什么？

通俗类比：Docker 就像集装箱。传统方式下，每个服务需要不同的运行环境（Go 需要 Go 运行时、Python 需要 Python 解释器……），就像货物没有集装箱，运输起来很麻烦。Docker 把"程序 + 运行环境"打包成一个"集装箱"（镜像）， anywhere 都能跑。

🍎 macOS 安装 Docker Desktop

  1. 访问 docker.com 下载 Docker Desktop for Mac
  2. 双击下载的 .dmg 文件，拖入 Applications 文件夹
  3. 在启动台（Launchpad）里找到 Docker 图标，点击启动
  4. 启动后，菜单栏右上角会出现一个鲸鱼图标 🐳，表示 Docker 正在运行

✅ 安装后验证

# 检查 Docker 版本
docker --version
# 预期输出：Docker version 26.x.x, build xxxxx

# 运行 hello-world 测试容器
docker run hello-world
# 预期输出：Hello from Docker! 表示安装成功
🖥️ 终端模拟器 — 预期输出
$ docker --version
Docker version 26.1.3, build 26.1.3-0ubuntu1~22.04.1

$ docker run hello-world

Hello from Docker!
This message shows that your installation appears to be working correctly.

To generate this message, Docker took the following steps:
1. The Docker client contacted the Docker daemon.
2. The Docker daemon pulled the "hello-world" image from the Docker Hub.
3. The Docker daemon created a new container from that image.
4. The Docker daemon streamed that output to the Docker client.

⚠️ 常见安装问题

  • Docker 启动后一直转圈：去"系统设置 → 隐私与安全"允许 Docker 内核扩展，然后重启 Docker
  • 报错 "Cannot connect to the Docker daemon"：Docker Desktop 没启动，去启动台打开它
  • Apple Silicon (M1/M2/M3) 芯片：下载时选择 "Apple Chip" 版本，不要用 Intel 版

安装 Minikube — 你的个人 Kubernetes 集群

🏠 为什么需要 Minikube？

通俗类比：Kubernetes（K8s）就像一个数据中心的管理系统，负责调度成百上千个容器。Minikube 就是在你自己的笔记本电脑里，虚拟出一个迷你数据中心，让你不用买云服务器就能学习 K8s。

📥 安装 Minikube

# 用 Homebrew 安装（推荐）
brew install minikube

# 验证安装
minikube version

了解 Kubernetes 基本操作

🎮 kubectl 是什么？

通俗类比：kubectl 就是 Kubernetes 的遥控器。就像用遥控器控制电视一样，用 kubectl 命令控制 K8s 集群里的各种资源（Pod、Service、Deployment 等）。

📚 核心概念速查

概念通俗解释类比
Pod一个运行中的容器实例一艘船上的一个集装箱
Deployment管理 Pod 的"蓝图"和副本数量造船厂的生产计划
Service为 Pod 提供稳定的网络入口港口的固定码头编号
Namespace集群内的"虚拟集群"，用于隔离资源船坞的不同区域

🔧 常用 kubectl 命令速览

# 查看所有 Pod（容器实例）
kubectl get pods

# 查看所有 Service（网络服务入口）
kubectl get services

# 查看 Pod 的详细信息和事件
kubectl describe pod 

# 查看 Pod 的运行日志
kubectl logs 

# 实时跟踪日志（类似 tail -f）
kubectl logs -f 

# 进入 Pod 里的容器（调试用）
kubectl exec -it  -- /bin/sh

# 查看集群节点状态
kubectl get nodes

# 查看所有资源（概览）
kubectl get all

📄 YAML 文件里有什么？

这个 YAML 文件里主要包含两种 K8s 资源：

  • Deployment：定义每个微服务需要多少个副本（Pod），用哪个 Docker 镜像，以及环境变量、资源限制等
  • Service：为每个微服务提供一个稳定的网络入口，让其他服务能通过服务名（如 currencyservice）访问它
  • ConfigMap / Secret：存储配置信息和敏感数据
  • PersistentVolumeClaim：为 Redis 提供持久化存储

🚨 如果 Pod 启动失败怎么办？

问题现象可能原因排查命令
STATUS = Pending资源不足（CPU/内存不够）kubectl describe pod <pod-name> 查看事件
STATUS = CrashLoopBackOff容器启动后马上崩溃kubectl logs <pod-name> 查看错误日志

访问前端页面 — 看到你的在线商店！

🌐 端口转发 — 把集群内的服务"暴露"到本地

由于 frontend 的 Service 类型是 ClusterIP（只在集群内可访问），我们需要用 kubectl port-forward 把集群内的 8080 端口转发到本地电脑的 8080 端口。

# 把 frontend 的 8080 端口转发到本地 8080
kubectl port-forward deployment/frontend 8080:8080
🖥️ 终端模拟器 — 端口转发
$ kubectl port-forward deployment/frontend 8080:8080
Forwarding from 127.0.0.1:8080 -> 8080
Forwarding from [::1]:8080 -> 8080
# 注意：这个命令会一直运行，不要按 Ctrl+C！ # 另开一个终端窗口来执行其他命令
⚠️ 重要提示：kubectl port-forward 会在前台持续运行，终端看起来像"卡住"了一样，这是正常的！不要按 Ctrl+C，直接打开浏览器访问即可。如需执行其他命令，请新开一个终端窗口。

🛍️ 打开浏览器访问

在浏览器中访问：http://localhost:8080

  • 商品列表：所有可购买的商品，每个商品显示图片、名称、价格
  • 商品详情页：点击商品后进入，显示详细描述、颜色/尺寸选项、数量选择
  • 购物车页面：点击购物车图标进入，显示已添加的商品列表和总价

💡 为什么用 Redis？

购物车数据需要快速读写（用户频繁加购/删购），而且不需要永久存储（用户退出后购物车可以清空）。Redis 是内存数据库，读写速度极快（毫秒级），非常适合这个场景。

类比：Redis 就像超市门口的存包柜，取放东西非常快，但空间有限（内存），适合临时存放。

使用负载生成器 — 模拟真实流量

🤖 负载生成器是什么？

通俗类比：负载生成器就像一个专业的"托儿"团队。他们模拟成百上千个真实用户，不停地浏览商品、加购物车、下单，用来测试系统在高并发下的表现。

🚀 理解微服务的弹性伸缩

在真实的 Kubernetes 生产环境中，我们可以配置 HPA (Horizontal Pod Autoscaler)，当 CPU 使用率超过某个阈值（如 70%）时，K8s 会自动增加该服务的 Pod 数量（比如从 1 个变成 3 个），实现"自动扩容"。

本项目为了简单，每个服务固定 1 个 Pod，但你可以尝试修改 YAML 文件里的 replicas 字段来体验多副本！

清理和后续学习

🧹 如何清理（停止并删除所有资源）

当你玩够了，想清理环境时：

# 删除所有部署的微服务（保留 Minikube 集群）
kubectl delete -f ./release/kubernetes-manifests.yaml

# 停止 Minikube 集群（保留集群配置，下次可以快速启动）
minikube stop

# 彻底删除 Minikube 集群（释放所有资源）
minikube delete
